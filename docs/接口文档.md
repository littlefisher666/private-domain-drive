# 私域网盘接口文档

## 1. 文档说明

- 项目名称：私域网盘
- 文档类型：一期接口文档
- 关联文档：
  - [PRD.md](/Users/littlefisher/Documents/LittleFisher/Workspaces/jyn/private-domain-drive/docs/PRD.md)

- 文档目标：定义 Flutter 客户端与轻后端之间的接口契约，以及客户端直连 OSS 前所需的能力与约束。

## 2. 设计范围

一期接口仅覆盖：

- 登录后获取 STS 临时凭证
- 获取基础配置
- 获取当前用户能力信息
- 健康检查与版本探测

不包含：

- 文件列表接口
- 上传接口
- 下载接口
- 删除接口

原因：

- 一期文件操作直接由客户端基于 STS 凭证访问 OSS
- 轻后端只承担鉴权、配置下发和少量控制面能力

## 3. 通用约定

### 3.1 协议约定

- 协议：HTTPS
- 数据格式：`application/json`
- 字符编码：UTF-8
- 鉴权方式：依赖上层登录态，后端根据登录态识别用户身份

### 3.2 域名与路径建议

- 基础路径：`/api/v1`
- 示例域名：`https://api.private-drive.example.com`

### 3.3 公共请求头

建议请求头：

```http
Content-Type: application/json
Accept: application/json
X-Platform: android | macos
X-App-Version: 1.0.0
X-Request-Id: <uuid>
```

### 3.4 公共响应结构

成功响应：

```json
{
  "code": "OK",
  "message": "success",
  "requestId": "8c6d7a66-b08e-4d78-98a7-0a3e7d3b8b8b",
  "data": {}
}
```

失败响应：

```json
{
  "code": "AUTH_REQUIRED",
  "message": "登录状态无效或已过期",
  "requestId": "8c6d7a66-b08e-4d78-98a7-0a3e7d3b8b8b"
}
```

### 3.5 错误码约定

| 错误码 | 含义 | 客户端处理建议 |
| --- | --- | --- |
| `OK` | 成功 | 正常处理 |
| `BAD_REQUEST` | 参数错误 | 提示请求异常，不自动重试 |
| `AUTH_REQUIRED` | 登录态失效 | 引导重新登录 |
| `FORBIDDEN` | 无权限访问 | 展示无权限提示 |
| `STS_ISSUE_FAILED` | 临时凭证签发失败 | 可重试，失败后提示稍后再试 |
| `CONFIG_NOT_FOUND` | 配置不存在 | 提示服务异常 |
| `RATE_LIMITED` | 请求过于频繁 | 延迟重试 |
| `INTERNAL_ERROR` | 服务内部异常 | 提示稍后重试 |

## 4. 接口列表

### 4.1 获取会话初始化信息

#### 接口说明

客户端登录成功后调用该接口，一次性获取：

- STS 临时凭证
- OSS 基础配置
- 当前用户身份
- 能力信息
- 上传与预览基础限制

#### 请求信息

- 方法：`POST`
- 路径：`/api/v1/session/bootstrap`

#### 请求体

```json
{
  "platform": "macos",
  "appVersion": "1.0.0",
  "deviceName": "MacBook-Pro",
  "capabilityVersion": "2026-06-01"
}
```

#### 字段说明

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `platform` | string | 是 | `android` 或 `macos` |
| `appVersion` | string | 是 | 客户端版本号 |
| `deviceName` | string | 否 | 设备名称，便于排查 |
| `capabilityVersion` | string | 否 | 客户端内置能力版本号 |

#### 成功响应

```json
{
  "code": "OK",
  "message": "success",
  "requestId": "9f970d9d-98a1-4fa6-861a-bf60a54b1633",
  "data": {
    "credentials": {
      "accessKeyId": "STS.xxxxxx",
      "accessKeySecret": "xxxxxx",
      "securityToken": "xxxxxx",
      "expiration": "2026-06-05T12:00:00Z"
    },
    "oss": {
      "bucket": "private-domain-drive",
      "region": "cn-hangzhou",
      "endpoint": "oss-cn-hangzhou.aliyuncs.com",
      "rootPrefix": "shared/"
    },
    "user": {
      "userId": "user-a",
      "displayName": "User A",
      "role": "member"
    },
    "capabilities": {
      "list": true,
      "download": true,
      "upload": true,
      "delete": false,
      "preview": true
    },
    "constraints": {
      "multipartUploadThresholdBytes": 10485760,
      "textPreviewMaxBytes": 524288,
      "allowedPreviewExtensions": [
        "jpg",
        "jpeg",
        "png",
        "gif",
        "pdf",
        "txt",
        "md"
      ]
    }
  }
}
```

#### 客户端处理要求

- 将 `credentials` 缓存在内存中
- 根据 `expiration` 提前刷新凭证
- 根据 `capabilities` 控制按钮显隐
- 根据 `constraints` 控制上传与预览行为

#### 失败场景

- 登录态失效：返回 `AUTH_REQUIRED`
- 用户未配置权限：返回 `FORBIDDEN`
- STS 调用失败：返回 `STS_ISSUE_FAILED`

### 4.2 刷新临时凭证

#### 接口说明

当客户端判断 STS 凭证即将过期，或访问 OSS 返回凭证失效时调用。

#### 请求信息

- 方法：`POST`
- 路径：`/api/v1/session/refresh`

#### 请求体

```json
{
  "platform": "android",
  "appVersion": "1.0.0"
}
```

#### 成功响应

```json
{
  "code": "OK",
  "message": "success",
  "requestId": "705d7c42-a086-4db0-a552-5ae550e950cb",
  "data": {
    "credentials": {
      "accessKeyId": "STS.newxxxx",
      "accessKeySecret": "newxxxx",
      "securityToken": "newxxxx",
      "expiration": "2026-06-05T13:00:00Z"
    }
  }
}
```

#### 处理建议

- 刷新成功后更新本地内存态
- 刷新失败且返回 `AUTH_REQUIRED` 时，清理会话并回到登录流程

### 4.3 获取当前用户能力

#### 接口说明

#### 请求信息

- 路径：`/api/v1/me/capabilities`

## 5. 客户端直连 OSS 约定

### 5.1 文件列表

客户端使用 `rootPrefix` 作为根目录范围，调用 OSS 列举能力。

建议返回字段映射：

| 客户端字段 | OSS 来源 |
| --- | --- |
| `path` | object key |
| `name` | key 最后一段 |
| `size` | object size |
| `updatedAt` | last modified |
| `isDirectory` | common prefix / key 规则 |
| `mimeType` | 根据扩展名推断 |

### 5.2 上传

- 小文件：直接上传对象
- 大文件：基于分片上传
- 上传对象 key 由客户端拼接当前目录前缀生成

建议对象 key 示例：

```text
shared/uploads/2026/06/example.jpg
```

### 5.3 下载

- 客户端直接从 OSS 下载到本地目录
- 文件名默认使用对象名
- 下载冲突时由客户端决定覆盖、重命名或提示

### 5.4 删除

- 客户端直接调用 OSS 删除
- 服务端不单独代理删除请求
- UI 不显示不代表安全，最终以 STS 权限为准

## 6. 时序建议

### 6.1 首次进入 App

1. 用户完成登录。
2. 客户端请求 `POST /api/v1/session/bootstrap`。
3. 获取 `credentials`、`oss`、`capabilities`、`constraints`。
4. 初始化 OSS 客户端。
5. 拉取文件列表并进入首页。

### 6.2 凭证续期

1. 客户端本地判断凭证接近过期。
2. 请求 `POST /api/v1/session/refresh`。
3. 刷新成功后替换内存中的凭证。
4. 刷新失败且登录态失效时回到登录页。

## 7. 安全要求

- 仅通过后端下发临时凭证，不向客户端暴露长期 AK/SK
- STS policy 必须限制到指定 bucket 和 prefix
- 所有接口走 HTTPS
- 接口层不返回多余敏感字段
- `requestId` 需贯穿日志，便于问题追踪

## 8. 后续可扩展接口

以下接口不属于一期必做，但后续可按需增加：

- 目录初始化接口
- 文件重命名接口
- 分享链接接口
- 用户可见空间配置接口
- 任务上报与错误统计接口

## 9. 结论

一期接口设计应保持极简，核心只保留“会话初始化 + 凭证刷新 + 能力与配置同步”。文件操作尽量由客户端携带 STS 直连 OSS，以维持低成本、低耦合和轻后端架构。
