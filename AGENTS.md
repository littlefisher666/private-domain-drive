# AGENTS.md

## 仓库定位

本仓库是 `Private Domain Drive` 项目的主仓库，职责是：

- 维护项目级文档
- 聚合客户端与服务端子仓库
- 记录架构、接口、模块拆分、协作约定

本仓库不是直接承载业务代码的主开发仓库。

## 当前目录结构

根目录下主要包含：

- `docs/`
- `private-domain-drive-client/`
- `private-domain-drive-server/`

其中：

- `docs/` 用于存放产品、技术、接口、模块设计等项目文档
- `private-domain-drive-client/` 是客户端独立子仓库
- `private-domain-drive-server/` 是服务端独立子仓库

## 子仓库职责边界

### 客户端子仓库

目录：`private-domain-drive-client/`

职责：

- Flutter 客户端开发
- Android 与 macOS 双端实现
- 页面、状态管理、上传下载、预览、平台适配等客户端逻辑

不应放入：

- 服务端函数代码
- 项目级非客户端文档

### 服务端子仓库

目录：`private-domain-drive-server/`

职责：

- 阿里云 FC 相关代码
- STS 凭证签发接口
- 配置下发、能力信息下发、健康检查等轻后端能力
- FC 部署配置、CI/CD 配置、Serverless Devs 配置

不应放入：

- Flutter 客户端代码
- 与服务端实现无关的项目杂项文件

## 主仓库内容边界

主仓库允许放入：

- `PRD`
- 技术方案文档
- 接口文档
- Flutter 模块拆分文档
- 架构图、流程图、部署说明
- 子仓库协作说明

主仓库默认不放入：

- 客户端业务代码
- 服务端业务代码
- 一次性构建产物
- 本地调试缓存文件

除非有明确说明，不要把客户端或服务端实现代码直接写到主仓库根目录或 `docs/` 以外的位置。

## 文档维护要求

所有跨仓库共享的信息，优先维护在主仓库 `docs/` 中，包括：

- 产品需求变化
- 技术架构调整
- 接口契约调整
- 客户端模块边界调整
- 服务端部署约束变化

当以下内容发生变化时，需要同步更新对应文档：

- 接口路径、字段、错误码变化：更新 `docs/接口文档.md`
- Flutter 分层、目录结构、模块职责变化：更新 `docs/Flutter模块拆分细节文档.md`
- 架构、技术选型、权限模型变化：更新 `docs/技术文档.md`
- 产品范围、目标、验收标准变化：更新 `docs/PRD.md`

## Git 与 Submodule 约定

本仓库使用 Git Submodule 管理两个子仓库。

要求：

- `private-domain-drive-client/` 作为独立 Git 仓库维护
- `private-domain-drive-server/` 作为独立 Git 仓库维护
- 主仓库只记录 submodule 引用版本，不在主仓库中直接替代子仓库提交历史

协作时注意：

- 修改客户端代码，应进入 `private-domain-drive-client/` 仓库提交
- 修改服务端代码，应进入 `private-domain-drive-server/` 仓库提交
- 子仓库提交更新后，如需在主仓库记录引用变化，再回到主仓库更新 submodule 指针

## 技术方向约定

当前项目的一期默认技术方向如下：

- 客户端：Flutter
- 服务端：阿里云函数计算 FC
- 服务端语言优先：Node.js
- 文件存储：阿里云 OSS
- 临时鉴权：阿里云 STS
- 轻量部署工具优先：Serverless Devs

对于服务端实现：

- 优先保持轻量
- 一期默认不引入重型 Web 框架
- 优先使用 FC handler + 轻量路由封装
- 文件上传下载不经服务端中转，客户端直连 OSS

## 开发约束

### 总体原则

- 优先做一期范围内的最小可用实现
- 不在未确认需求时提前扩展二期能力
- 不为“看起来更完整”而引入重型基础设施
- 不轻易增加数据库、常驻服务器、复杂后台

### 客户端约束

- Android 与 macOS 共用大部分业务逻辑
- 平台差异尽量收敛在基础设施适配层
- 目录结构、分层设计优先遵循 `docs/Flutter模块拆分细节文档.md`

### 服务端约束

- 服务端职责仅限轻量鉴权与配置能力
- 一期不承担文件流量中转
- 接口设计优先遵循 `docs/接口文档.md`

## 命名与组织约定

- 仓库名统一使用 `private-domain-drive-*`
- 文档文件名可使用中文，但需要语义明确
- 新增项目级文档优先放在 `docs/`
- 不新增无明确用途的一次性脚本文件

## 协作建议

开始新工作前，优先判断修改应发生在哪一层：

- 产品或架构变更：主仓库 `docs/`
- Flutter 客户端实现：`private-domain-drive-client/`
- FC 服务端实现：`private-domain-drive-server/`

如果一个改动同时影响文档与代码：

1. 先更新设计和接口文档
2. 再更新对应子仓库实现
3. 最后在主仓库记录必要的 submodule 版本变更

## 提交前检查

提交前建议确认：

- 是否把代码放到了正确的仓库
- 是否同步更新了对应文档
- 是否误把构建产物或本地缓存提交进仓库
- 是否改动了 submodule 但没有意识到主仓库指针已变化
